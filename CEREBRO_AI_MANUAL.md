### 4.4 전처리
- 선택한 데이터로부터 전처리 과정을 거친 데이터를 조회 및 분석을 하기 위함
#### 4.4.1 전처리 설정
<img src="./4_분석/분석_전처리_전처리설정.png">

#### ① AI 전처리 
- AI가 현재 단계의 데이터를 분석하여 전처리를 제안하고 전처리 목록을 추가
  <img src="./4_분석/분석_전처리_설정_AI전처리.png">
  ##### ⓐ AI 전처리 버튼
  - 버튼을 누르면 AI가 현재 단계 데이터를 분석하여 ⓒ 과 같이 나오며, ⓔ에 전처리 목록을 등록

  ##### ⓑ AI 전처리 요청 창 생성 및 숨기기 버튼
  - 버튼을 누를 시 AI 전처리 직접입력창이 ⓓ와같이 생성

  ##### ⓒ AI 전처리 제안 조회
  - AI가 분석한 전처리 제안 조회

  ##### ⓓ AI 전처리 요청 직접입력창
  - LLM을 통해 요청의 목적을 이해하고 전처리 과정을 진행
  - 요청이 불명확할 시 전처리 함수를 호출하지 않는다.

| 전처리설정 | 설명 |
|---|---|
|결측값 대치 방식|simple(평균·중앙값·최빈값처럼 단순 통계값으로 결측값을 채우는 방식)<br>Iterative(다른 변수들과의 관계를 학습하여 예측값으로 반복적으로 결측값을 채우는 방식)|

#### ④ 학습 알고리즘 선정
- 모델 학습에 사용할 알고리즘 선택

##### 1. 항목 내역

| 항목 | 설명 |
|---|---|
| 알고리즘 | 모델 학습 시 사용할 알고리즘 유형<br> (하단 알고리즘 참고) |
| R2 | 모델의 데이터 설명력을 나타내는 적합도 지표 |
| RMSE | 예측값과 실제값의 차이를 기반으로 계산한 오차 지표 |
| RMSLE | 실제값과 예측값의 로그 차이를 기반으로 계산한 오차 지표 |
| MAPE | 실제값 대비 예측 오차의 평균 비율(%)을 나타내는 지표 |
| Accuracy | 전체 데이터 중 정확하게 분류한 비율 |
| AUC | ROC 곡선 아래 면적으로 모델의 분류 성능을 평가하는 지표 |
| Recall | 실제 양성 데이터 중 올바르게 예측한 비율 |
| Prec. | 양성으로 예측한 데이터 중 실제 양성 비율 |
| F1 | Precision과 Recall의 조화 평균으로 계산한 성능 지표 |
| Kappa | 실제값과 예측값 간 일치도를 우연성을 제외하고 평가하는 지표 |
| MCC | 정분류와 오분류를 모두 고려한 이진 분류 성능 지표 |

- 보고서 작성을 위한 주제를 선정하는 기능
- 업로드 된 데이터와 어느정도의 주제가 일치해야 함.
- 보고서 작성 톤과 분량을 지정

##### 7.3.1.2 데이터셋 선택

<img src="./7_보고서/보고서_AI보고서만들기_데이터셋 지정.png">

- 작성할 보고서의 주요 데이터셋 지정
- 업로드 된 데이터를 기반으로 작성

##### 7.3.1.3 위젯추가

<img src="./7_보고서/보고서_AI보고서만들기_위젯추가.png">

- 작성한 주제와 선택한 데이터를 기반으로 AI가 분석하여 자동으로 위젯을 생성
- 주제와 관련이 높은 신뢰도를 바탕으로 정렬하며 채택을 통해 본문 보고서에 추가
- 추천 다시 받기를 통하여 재생성 하며, 3개씩 AI가 생성


##### 7.3.1.4 기존 대시보드 위젯 추가

<img src="./7_보고서/보고서_AI보고서만들기_기존대시보드추가.png">

- 기존에 작성된 대시보드나 위젯을 선정하여 본문에 추가하는 기능
- 없을 경우 스킵가능

##### 7.3.1.5 본문생성

<img src="./7_보고서/보고서_AI보고서만들기_본문생성.png">

- 작성된 주제와 선택한 데이터를 바탕으로 보고서 본문 작성
- 보고서 저장을 통해 생성되며, 최신화를 통해 위젯 활성화
